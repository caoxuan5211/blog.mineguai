hugo项目结构#

my-blog/
├── archetypes/         # 内容模板
├── assets/             # 需要 Hugo Pipes 处理的资源 (SCSS, JS)
├── content/            # Markdown 内容
├── layouts/            # 模板布局 (覆盖主题)
├── static/             # 静态文件 (图片, 字体)
├── themes/             # 主题目录
└── hugo.yaml           # 核心配置文件

快速生成并推送到github网页#

1.Windows系统#

编辑一个deploy.ps1文件(.bat 是 cmd 的脚本，而 .ps1 是 PowerShell 的脚本)实现自动化推送。下面示例统一使用 main 作为部署分支：

 1param (
 2    [string]$msg = $(Read-Host "请输入本次提交信息")
 3)
 4
 5$deployBranch = "main"
 6
 7# 先切到部署分支，确保 public/ 统一使用主分支
 8Set-Location F:\myblog\public
 9git switch $deployBranch
10
11# 进入 Hugo 项目根目录并生成网页
12Set-Location F:\myblog
13hugo -D --cleanDestinationDir
14
15# 进入 public/ 目录
16Set-Location F:\myblog\public
17
18# 添加构建产物
19git add .
20
21git diff --cached --quiet
22if ($LASTEXITCODE -ne 0) {
23    git commit -m "$msg"
24} else {
25    Write-Host "没有可提交的构建变更" -ForegroundColor Yellow
26}
27
28# 强制推送到 main 分支
29git push -f origin $deployBranch
30
31# 输出提示信息
32Write-Host ""
33Write-Host "======================================" -ForegroundColor Yellow
34Write-Host "✅ 网站已成功部署！" -ForegroundColor Green
35Write-Host "🌐 访问地址: https://blog.mineguai.com/" -ForegroundColor Cyan
36Write-Host "🌿 部署分支: main" -ForegroundColor Cyan
37Write-Host "======================================" -ForegroundColor Yellow

hugo“模板覆盖”规则#

Hugo 会先在我们自己的站点根目录 layouts/ 下查找模板文件，如果存在就用它，而不用 themes/PaperMod/layouts/ 的文件（被覆盖了）。

例如：

themes/PaperMod/layouts/partials/head.html  # papermod主题自带
myblog/layouts/partials/head.html       # 我们自己的的覆盖文件，会优先使用

只要你在 myblog/layouts/partials/ 里放同路径的文件，Hugo 就会用你自己的版本。
